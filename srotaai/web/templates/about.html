What SrotaAI does

Adverse drug reactions kill more Indians every year than dengue and malaria combined — yet over 96 % go unreported through formal CDSCO channels. Patients post about side-effects on Reddit, 1mg, ehealthforum and WhatsApp groups in real time, while regulators see only a thin trickle of formal MedWatch / FAERS forms months later.

SrotaAI is the bridge. It listens to where patients actually talk, extracts drug↔reaction pairs with NLP, computes statistical disproportionality (PRR, ROR, χ², IC) on rolling windows, and surfaces signals with auditable provenance — turning months-late paperwork into days-fresh intelligence.

1 Multi-source ingestion

  • ● openFDA / FAERS — live api.fda.gov/drug/event.json
  • ● FDA Recalls RSS — real-time recall alerts
  • ● Reddit — r/india, r/IndianMedicine (anonymous JSON or PRAW)
  • ● HTML stealth-scrape — 1mg, ehealthforum reviews
  • ○ WhatsApp Business — Meta Cloud API (webhook ready)

2 NLP & entity extraction

  • • Deduplication via SHA-256 record hash
  • • PII redaction pipeline (phone, email, Aadhaar)
  • • Drug + symptom NER with ~50 generic drugs and expanded symptom dictionary
  • • Hindi/Hinglish keyword expansion (e.g. khujli → itching, daane → rash)
  • • Sentiment classification (ADR / negative / neutral / positive)
  • • Drug↔event pair linking per record

3 Statistical signal detection

  • • Industry-standard 2×2 disproportionality:
  • PRR = (a/(a+b)) ÷ (c/(c+d))
  • ROR = (a·d) ÷ (b·c)
  • χ² with Yates correction
  • IC = log₂(observed / expected)
  • • Tiered: Critical (PRR≥3), Watch (2≤PRR<3), Monitor
  • • Confidence weighted by sample size and source quality

4 Audit-grade compliance

  • • Every action recorded in audit_log
  • • Hash-chained events — tamper-evident à la blockchain
  • • Immutable project configuration after creation
  • • Source provenance preserved on every signal
  • • CSV export for regulator submissions
  • • Compatible with CDSCO PvPI and ICH E2B(R3) workflow
